## Decisões principais

### 1. Escudo por MÁSCARA (`mask-image`), não clip-path nem SVG inteiro
O SVG vira **máscara alfa** de um fundo (`background`). Isso desacopla **forma** de **conteúdo**: o mesmo recorte serve para cores (sólido/listras/grade/bola) **ou** foto. `mask-mode: alpha` explícito por robustez cross-browser (WebKit já usa alpha por padrão; Chrome/Firefox honram o valor).

### 2. Catálogo de formas via `import.meta.glob`, em `src/assets`
A lista de formas é montada automaticamente a partir de `src/assets/escudos/*.svg` e `src/assets/federacoes/*.svg`. Gerenciar = largar/remover/renomear arquivo. **O Vite não enumera `public/`**, então os SVGs foram movidos para `src/`. SVGs pequenos viram data URI no bundle (funcionam igual como `mask-image`). Há `README.md` em cada pasta com a convenção (`escudo-<id>.svg` / `flamula-<id>.svg`) — não renomear arquivos em uso (o `<id>` fica salvo no perfil/federação).

### 3. Todo perfil tem escudo — normalização no RENDER, sem migration destrutiva
`legacyToCrest()` converte qualquer `avatar_url` antigo: `crest:` passa direto; `gen:` (avatar antigo) vira escudo com as cores dele; foto crua (URL do Google salva direto) vira escudo recortando a foto; `null` cai no escudo padrão do nome. Removido o caminho que renderizava foto redonda "sem escudo". Zero alteração no banco — perfis antigos passam a exibir escudo automaticamente.

### 4. `Button` com `type="button"` por padrão
`<button>` sem `type` é `submit`. Dentro do `<form>` do editor de perfil, "Sortear" (e qualquer `Button`) enviava o formulário e salvava. Corrigido no componente base; os botões de envio reais já declaram `type="submit"`.

### 5. Dados de jogos: ESPN (descartados scraping Python e API-Football)
- **Scraping (ex.: `soccerdata`)**: Python, agrega HTML (FBref/Sofascore/FotMob), exige worker separado + quebra com mudança de página + anti-bot. Descartado.
- **API-Football (api-sports.io)**: dado ótimo (status real, logos), mas o **plano Free é travado em temporadas 2022–2024** — **não dá acesso a 2026**. Descartado para os amistosos atuais. (A chave que o João criou **não foi armazenada em lugar nenhum** — nenhum segredo configurado.)
- **ESPN** (`site.api.espn.com/.../soccer/{liga}/scoreboard?dates=YYYYMMDD`): JSON público, grátis, com status (pre/in/post + canc/postp), placar, **escudos**, nomes e um `calendar` com todas as datas que têm jogo. Roda direto na edge function Deno, sem infra nova. **Escolhido.**

### 6. Curadoria por jogo via `matches.hidden` + filtro client-side resiliente
Coluna `hidden` (jogo oculto não entra pra palpitar). O filtro no `useMatches` é **client-side** (`!m.hidden`) de propósito: se a coluna chegar no deploy com atraso, `m.hidden` é `undefined` e nada quebra na tela de jogos. A escrita já é coberta pela policy `matches_admin_write` (app_admin = ALL), então não precisou de RPC nova.

### 7. "Todos os campeonatos" como padrão + reset de dia ao trocar escopo
O dia selecionado **vazava** entre competições (mantinha o dia anterior se ele existisse na nova lista) — por isso "Amistosos" abria num dia distante. Agora o dia **reseta** ao trocar de escopo e recalcula: hoje → senão o próximo com jogos → senão o último.

### 8. Numeração de migration entre sessões paralelas — a armadilha da sessão
A sessão `feat/confrontos` e esta criaram, cada uma, uma migration `20260603000011_*`. O Supabase controla migrations **por número de versão**, então a `011` já constava como aplicada (pela do confronto) e a do `espn` foi **pulada** — o enum `data_provider` não recebia `'espn'` em produção (erro ao adicionar competição ESPN). Corrigido renumerando para `...014`. **Lição:** com trabalho em paralelo, combinar faixas de numeração ou sempre conferir o último número antes de criar.

## Cuidados aprendidos (referência rápida)
- **Colisão de número de migration** entre sessões paralelas faz o Supabase **pular** a 2ª com o mesmo número. Combinar faixas / conferir o último número.
- **Vite não enumera `public/`** — assets que precisam ser listados (glob) vão em `src/`.
- **`<button>` sem `type` dentro de `<form>` = submit.** O componente base deve assumir `type="button"`.
- **`PostgrestError` não é `Error`** — relançar `new Error(error.message)` para o toast não engolir como "Erro.".
- **API-Football Free** é travado em temporadas antigas (2022–2024); não serve para dado atual.
- **Testar com dado real ANTES de construir** — testar o API-Football antes evitou construir um provedor inútil para 2026.
- Chave de API enviada pelo usuário: **não armazenar em código nem em segredo sem necessidade** (a do API-Football não foi usada/guardada).
